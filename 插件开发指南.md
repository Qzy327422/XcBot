XcBot 插件开发指南

XcBot 支持从 plugins/ 目录热加载外部插件，仅需少量代码即可快速扩展功能。

插件放置位置：
- 单文件插件：plugins/hello.py
- 目录插件（推荐复杂功能）：plugins/hello_plugin/setup.py

最小插件示例：

TRIGGHT_KEYWORD = "你好世界"
HELP_MESSAGE = "你好世界 → 返回一句问候"

async def on_message(event, actions, Manager, Segments):
    await actions.send(
        group_id=event.group_id,
        message=Manager.Message(Segments.Text("Hello, world! 🌍"))
    )
    return True

必须包含的内容：
- TRIGGHT_KEYWORD（注意拼写，必须是 TRIGGHT_KEYWORD，不是 TRIGGER_KEYWORD）
- 异步函数 on_message

触发机制：
- 普通插件：当消息中包含 TRIGGHT_KEYWORD 时触发（适合命令式功能，如 /echo）
- 推荐使用短小清晰的关键词，避免与其他功能冲突（如 签到、抽卡）

on_message 参数说明：

常用写法：
async def on_message(event, actions, Manager, Segments, order="", is_group=False, user_message=""):
    ...

常用可用参数：
- event：消息事件对象（包含 group_id、user_id 等）
- actions：用于发送消息、撤回消息等操作
- Manager、Segments：消息构造工具
- order：命令后面的参数
- is_group：是否为群聊
- user_message：原始消息文本

建议：不确定的参数请加上默认值，避免运行时报错。

返回值规则：
- return True：插件已完全处理该消息，宿主停止后续流程
- return False 或不返回：插件未完全接管，继续执行后续逻辑（包括 AI 回复）

目录插件：
结构示例：
plugins/
└── my_plugin/
    └── setup.py

目录插件的 setup.py 写法与单文件插件一致。

Any 插件（全局监听）：

TRIGGHT_KEYWORD = "Any"

适合做敏感词过滤、前置判断等功能。使用时需谨慎，避免误拦截正常消息。

示例：
TRIGGHT_KEYWORD = "Any"

async def on_message(user_message=""):
    if "违禁词" in user_message:
        # 可执行撤回等操作
        return True
    return False

插件管理命令：
- /重载插件 → 重新从磁盘加载所有插件
- /禁用插件 插件名 → 临时禁用（文件会自动重命名为 d_xxx）
- /启用插件 插件名 → 重新启用
- /插件视角 → 查看已加载、已禁用、加载失败的插件列表

最佳实践与常见问题：

最佳实践：
- 插件内部建议添加 try...except 异常处理，避免影响主程序运行
- 复杂功能推荐使用目录插件 + setup.py
- 开启外部插件开关：
  FeatureSwitches: {
      "plugins_external": true
  }

常见问题：

1. 插件未加载：
   - 检查 FeatureSwitches.plugins_external 是否为 true
   - 插件文件是否放在 plugins/ 目录下
   - 目录插件是否正确包含 setup.py
   - 文件是否被重命名为 d_ 开头（禁用状态）

2. 写了 TRIGGER_KEYWORD 但无效：
   项目实际使用的是 TRIGGHT_KEYWORD（注意多一个 H）

3. 缺少参数报错：
   为参数添加默认值，或者删除不需要的参数。

4. 插件报错导致主流程异常：
   在插件内部自行捕获异常，并返回 False。
